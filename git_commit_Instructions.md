# Git Commit Message 指引

根據程式碼變更內容，自動撰寫 Git Commit Message，請遵守以下規範。

## 輸出格式規範

1. 全文使用繁體中文
2. 每個分類以「--- 類別 ---」作為分隔線
3. 每個分類下以條列式列出項目（使用 1. 2. 3. 4. 開頭）
4. 每項描述請以動詞開頭，且控制在 50 字內
5. 若該分類無項目，請略過該分類

## Commit Message 分類

| 分類 | 說明 |
|------|------|
| **feat** | 新增或修改功能 |
| **fix** | 修補錯誤 |
| **docs** | 文件內容更新 |
| **refactor** | 重構 (非新功能或修正 bug) |
| **perf** | 改善效能 |
| **test** | 增加或修改測試 |
| **chore** | 建構程序或輔助工具變動 |
| **revert** | 撤銷先前的 commit |

## 分類判斷邏輯

- **新增函式、API、UI、業務邏輯** → `feat`
- **修正例外處理、錯誤邏輯、錯字** → `fix`
- **僅修改文件、README、註解** → `docs`
- **僅重構代碼結構、命名、抽取共用函式** → `refactor`
- **調整演算法、減少運算時間** → `perf`
- **新增或修改單元測試、整合測試** → `test`
- **修改建置腳本、設定檔、CI/CD** → `chore`
- **回覆或撤銷前一次提交** → `revert`

## 輸出範例

```
--- feat ---
1. 新增使用者登入驗證模組
2. 調整前端 API 呼叫邏輯

--- fix ---
1. 修正登入頁面未正確顯示錯誤訊息
2. 修補 NullReferenceException 問題

--- refactor ---
1. 重構資料存取層以提高可維護性
```
